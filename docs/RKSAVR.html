Class: RKSAVR

RKSAVR(config)

Randomized Kolmogorov-Smirnov Analysis of Volatility Roughness estimator. The class encapsulates a *configurable* estimator instance: the scales to analyze, the sampler used inside each variance-reduction iteration, the 1D optimizer that minimizes the KS distance surface, and the safe bounds the final estimate will be clamped to. Lifecycle: 1. The constructor stores the configuration and pulls a fresh optimizer function from optimization/registry. The estimator is therefore stateful across calls — each call to RKSAVR#estimate draws a new independent PRNG state via `prng.js`. 2. RKSAVR#estimate averages `iterations` independent RKSAVR#estimateSingle results for variance reduction. 3. RKSAVR#rolling and RKSAVR#rollingMultiScale are convenience wrappers around a sliding window of these estimates. 4. RKSAVR#estimateBatch performs non-overlapping window estimation for parallel processing pipelines. Thread-safety: the estimator does not share mutable state between its methods (no internal counters) but it does rely on the *global* seeded PRNG exposed by prng. Callers that need concurrent estimation should serialize the calls (e.g. wrap in an async queue) or seed the PRNG with a per-call seed. Example: ```javascript import {RKSAVR} from 'rksavr'; const estimator = new RKSAVR({scaleA1: 1, scaleA2: 50, sampleSize: 500, iterations: 16}); const H = estimator.estimate(logVolSeriesWindow); ```

Constructor

new RKSAVR(config)

Builds an RKSAVR estimator with the given configuration. All parameters are optional; sensible defaults match the paper's recommended settings for daily equity log-volatility. Pass an explicit `iterations: 1` to disable variance reduction and a non-null `blockSize` to enable paper-faithful block random permutation.

estimate(window, opts) → {number}

Variance-reduced estimate: averages `iterations` independent single-shot estimates. Returns the mean of the successful iterations and logs the failed ones through the leveled logger. If every iteration fails (extremely unlikely for healthy inputs) the method throws so callers are not silently handed back `0` or `NaN`.

rolling(series, windowSize, step, onProgress) → {Array.<{t: number, H: ?number, error: ?string}>}

Sliding-window estimation along a long series. For each anchor `t` the method slices `series[t : t + windowSize]` and runs `estimate()` on the slice. Failures are captured into the result as `{t, H: null, error}` instead of aborting the loop. Performance: each call to `estimate()` re-uses the configured variance-reduction iterations; consider shrinking `iterations` for monitoring workloads.

rollingMultiScale(series, windowSize, scales, weightsopt, step, onProgress) → {Array.<{t: number, H: ?number, profile: Array.<number>, error: ?string}>}

Multi-scale rolling estimation that exposes the KS profile per window. For each anchor `t` the method performs a **one-shot** multi-scale estimate (no variance reduction — this method favors throughput over stability) and additionally returns the flat `profile` of pairwise KS distances at the optimal `H`. The profile is a `|scales| * (|scales| - 1) / 2` array suitable for diagnostic plotting.

(static) getIncrements(series, scale) → {Float64Array}

Computes the increments of a series at a single scale. For each index `i` returns `series[i + scale] - series[i]`. The result has length `series.length - scale`. Inputs may be a regular array or a `Float64Array`. The implementation is allocation-friendly: the output is always a `Float64Array`. Complexity: O(n).
